Cycling Power Service Library

While working on my Autoshift project, I ran into an issue. Currently, there is no library to interpret data from a BLE power meter. This problem was because nobody had dealt with microcontrollers and power meters using CircuitPython. Because of this I couldn't receive data from my power meter and couldn't find a way to do so without creating my own service library. I started working on this project and got deeply involved with learning about BLE connections and how the service works. I worked on the library to be able to use direct data from my power-meter. Throughout the process, I worked with developers at Adafruit and other people who have worked on CircuitPython libraries in the past to get my library added to the CircuitPython_Community Bundle. The service was implemented into the AutoShift project and worked flawlessly with the Sram AXS power-meter. This project ended up being a great success and allowed me to further my knowledge of BLE packets and how to extract necessary data from the packet. This project helped spark my interest in wireless communications.

Key Features

  • Return Power Value: Returns the power value from the sensor in Watts.
  • Return Byte Array: Gives the user the option to return the entire byte array sent from the powermeter so that they can extract what data they would like to from the sensor.

Technologies Used

  • BLE: Uses the Adafruit-ItsyBitsy-NRF52840 to connect to the sensor and then read the data.
